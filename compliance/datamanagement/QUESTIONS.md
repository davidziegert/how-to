# Fragenkatalog zur Erstellung eines Datenmanagementplans (DMP)

## 1. Projektbeschreibung und Ziele

- Was ist das Ziel des Forschungsprojekts?
- Welche Forschungsfragen sollen beantwortet werden?
- Welche Rolle spielen Daten im Projekt?
- Welche Arten von Forschungsdaten werden im Projekt erzeugt oder genutzt?

## 2. Datentypen und Datenformate

- Welche Datentypen entstehen im Projekt (z. B. Umfragedaten, Interviews, Logdaten)?
- Handelt es sich um strukturierte, unstrukturierte oder semi-strukturierte Daten?
- Welche Datenformate werden verwendet?
- Werden offene und langfristig nutzbare Formate eingesetzt?
- Welche Software oder Tools werden zur Verarbeitung der Daten benötigt?

## 3. Datenerhebung und Datenquellen

- Wie werden die Daten erhoben (z. B. Umfrage, Experiment, Messung)?
- Welche Datenquellen werden genutzt?
- Werden vorhandene Datensätze oder externe Daten integriert?
- Wie wird sichergestellt, dass die Datenerhebung ethischen und rechtlichen Anforderungen entspricht?

## 4. Organisation, Dokumentation und Metadaten

- Wie werden die Daten organisiert und strukturiert?
- Welche Metadaten werden erfasst?
- Welche Dokumentation begleitet die Datensätze (z. B. README, Codebooks)?
- Wie wird sichergestellt, dass andere Forschende die Daten verstehen und nachnutzen können?
- Wird eine Versionierung der Daten und Analyse-Skripte durchgeführt?

## 5. Datenspeicherung und Datensicherheit

- Wo werden die Daten während des Projekts gespeichert?
- Welche Maßnahmen schützen die Daten vor Verlust oder unbefugtem Zugriff?
- Wie werden Backups durchgeführt?
- Wer hat Zugriff auf die Daten?
- Wie werden sensible oder personenbezogene Daten geschützt (z. B. Verschlüsselung, Pseudonymisierung)?

## 6. Sicherstellung der Datenqualität

- Welche Maßnahmen stellen sicher, dass die Daten korrekt und vollständig sind?
- Werden Validierungs- oder Plausibilitätsprüfungen durchgeführt?
- Wie werden Fehler oder Inkonsistenzen in Datensätzen erkannt und korrigiert?
- Wie wird die Datenqualität während des Projekts überwacht?

## 7. Umsetzung der FAIR-Prinzipien

Findable (Auffindbar)

- Werden die Daten in einem Repositorium veröffentlicht?
- Erhalten die Datensätze einen persistenten Identifikator (z. B. DOI)?
- Welche Metadaten unterstützen die Auffindbarkeit der Daten?

Accessible (Zugänglich)

- Wer darf auf die Daten zugreifen?
- Werden die Daten öffentlich zugänglich sein oder nur eingeschränkt verfügbar?
- Welche Zugriffsbedingungen gelten?

Interoperable (Interoperabel)

- Werden standardisierte Datenformate und Metadaten verwendet?
- Können die Daten problemlos mit anderen Systemen oder Datensätzen kombiniert werden?

Reusable (Wiederverwendbar)

- Sind die Daten ausreichend dokumentiert?
- Welche Lizenzbedingungen gelten für die Datennutzung?
- Können andere Forschende die Daten problemlos wiederverwenden?

## 8. Rollen und Verantwortlichkeiten

- Wer ist für das Datenmanagement im Projekt verantwortlich?
- Gibt es definierte Rollen (z. B. Data Steward, Data Owner)?
- Wer kümmert sich um Datensicherung, Dokumentation und Veröffentlichung?

## 9. Datenlebenszyklus

- Wie werden die Daten geplant, erhoben, verarbeitet und analysiert?
- Wie werden Daten während des Projekts verwaltet?
- Wie werden Daten nach Abschluss des Projekts archiviert oder gelöscht?
- Welche Schritte stellen die langfristige Verfügbarkeit sicher?

## 10. Archivierung und Langzeitverfügbarkeit

- Wo werden die Daten langfristig archiviert (z. B. Forschungsdatenrepositorium)?
- Wie lange sollen die Daten aufbewahrt werden?
- Werden offene Formate verwendet, um die langfristige Nutzbarkeit sicherzustellen?
- Welche Daten werden archiviert und welche ggf. gelöscht?

## 11. Rechtliche und ethische Aspekte

- Enthalten die Daten personenbezogene oder sensible Informationen?
- Wie wird der Datenschutz sichergestellt (z. B. DSGVO)?
- Liegen Einwilligungen der Teilnehmenden vor?
- Welche rechtlichen Einschränkungen bestehen bei der Veröffentlichung der Daten?
